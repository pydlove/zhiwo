# 微信公众号头部博主整理小程序 - 设计文档

## 1. 产品定位

### 1.1 目标用户
创作者（公众号运营者、自媒体新人），需要快速找到所在赛道的头部博主，查看其作品进行对标学习。

### 1.2 核心场景
用户打开小程序 → 看到热门赛道排行 → 点击自己的赛道 → 看到该赛道 TopN 头部博主 → 点击博主 → 看到其代表作品标题和链接 → 复制到微信搜索/浏览器查看原文。

### 1.3 MVP 原则
- 只做一个平台：公众号
- 只做内容展示 + 复制功能
- 无用户登录、无UGC、无搜索、无推荐算法
- 底部仅保留一个原生 tab：「公众号」

## 2. 页面结构

### 2.1 页面总览
| 页面 | 路径 | 说明 |
|------|------|------|
| 首页 | `/pages/index/index` | 热门赛道排行列表，同时也是唯一的 tab 页 |
| 赛道页 | `/pages/track/track?id={trackId}` | 某赛道的 TopN 博主列表 |
| 博主作品页 | `/pages/blogger/blogger?id={bloggerId}` | 某博主的代表作品清单 |

### 2.2 首页（index）
- **标题栏**：小程序默认导航栏，标题为「热门赛道」
- **内容区**：
  - 页面标题：🔥 热门赛道（16px，加粗）
  - 赛道列表：纵向卡片列表，按热度/排序字段排列
    - 每个卡片展示：
      - 左侧排名数字（大号、加粗，前三名用橙色/金色系，其余用灰色）
      - 赛道名称 + emoji 图标（14px，加粗）
      - 预览文字：该赛道前 3 个博主名称（11px，灰色）
      - 右侧操作文字：查看 →（绿色）
    - 卡片可点击，点击后跳转赛道页
- **底部**：流量主 banner 广告位
- **原生 tabBar**：仅一个 tab「公众号」

### 2.3 赛道页（track）
- **标题栏**：动态显示赛道名称，如「🎬 个人IP」
- **内容区**：
  - 页面标题：赛道名称（14px，加粗）
  - 博主列表：纵向卡片列表，按排名正序排列
    - 每个卡片展示：
      - 左侧排名数字（圆形背景，36x36px，12px 字）
      - 博主名称（13px，加粗）
      - 一句话定位（11px，灰色），如「写作课创始人 · 10w+阅读」
      - 右侧操作文字：查看作品 →（绿色）
    - 卡片可点击，点击后跳转博主作品页
- **底部**：流量主 banner 广告位
- **返回**：点击左上角返回按钮回到首页

### 2.4 博主作品页（blogger）
- **标题栏**：博主名称
- **内容区**：
  - 博主信息区：
    - 头像（48x48px，圆形，占位灰色）
    - 博主名称（15px，加粗）
    - 所属赛道 + 排名（11px，灰色），如「个人IP赛道 · Top 1」
  - 作品列表标题：🔥 代表作品（12px，加粗）
  - 作品列表：纵向卡片
    - 每个卡片展示：
      - 作品标题（13px，深黑色）
      - 两个操作按钮（11px，灰色背景圆角按钮）：
        - 🔗 复制链接
        - 📋 复制标题
    - 点击「复制链接」：调用 `wx.setClipboardData` 复制文章链接
    - 点击「复制标题」：调用 `wx.setClipboardData` 复制文章标题
- **底部**：流量主 banner 广告位
- **返回**：点击左上角返回按钮回到赛道页

## 3. 数据模型

### 3.1 赛道（Track）
```json
{
  "id": "personal-ip",
  "name": "个人IP",
  "icon": "🎬",
  "sortOrder": 1,
  "previewBloggers": "粥左罗、剽悍一只猫、Spenser"
}
```

### 3.2 博主（Blogger）
```json
{
  "id": "zhouzuoluo",
  "name": "粥左罗",
  "avatar": "https://...",
  "tagline": "写作课创始人 · 10w+阅读",
  "trackId": "personal-ip",
  "rank": 1
}
```

### 3.3 作品（Post）
```json
{
  "id": "post-001",
  "title": "普通人如何靠写作年入百万",
  "url": "https://mp.weixin.qq.com/s/xxxxx",
  "bloggerId": "zhouzuoluo"
}
```

## 4. 交互规范

### 4.1 复制反馈
- 调用 `wx.setClipboardData` 成功后，显示 `wx.showToast({ title: '已复制', icon: 'success', duration: 1500 })`

### 4.2 页面跳转
- 首页 → 赛道页：`wx.navigateTo({ url: '/pages/track/track?id=' + trackId })`
- 赛道页 → 博主页：`wx.navigateTo({ url: '/pages/blogger/blogger?id=' + bloggerId })`

### 4.3 空状态
- MVP 阶段暂不考虑空状态，上线时确保每条数据都有完整内容。

## 5. 视觉规范

### 5.1 配色
- 主背景色：`#f5f5f5`（浅灰）
- 卡片背景：`#ffffff`（白色）
- 主文字：`#1a1a1a`（深黑）
- 次要文字：`#555555`（深灰）
- 辅助文字：`#666666`（中灰）
- 操作强调色：`#07c160`（微信绿）
- 排名 Top1：`#ff4d4f`
- 排名 Top2：`#ff7a45`
- 排名 Top3：`#ffa940`

### 5.2 字体
- 标题/名称：系统默认字体，14-16px，font-weight: 600
- 正文/描述：系统默认字体，12-13px，font-weight: 400
- 辅助信息：系统默认字体，11px，font-weight: 400

### 5.3 间距
- 页面左右边距：16px
- 卡片之间间距：10px
- 卡片内部间距：12-14px
- 按钮圆角：4px
- 卡片圆角：8-10px

---

## 6. 流量主变现

- 首页底部：banner 广告（`ad-unit-id` 占位）
- 赛道页底部：banner 广告
- 博主作品页底部：banner 广告

---

## 7. 技术选型

- **框架**：微信小程序原生开发（WXML + WXSS + JS）
- **不使用**第三方 UI 框架，保持最小依赖
- **数据存储**：MVP 阶段使用本地 `data` 硬编码或 `utils/data.js` 模块
- **后续扩展**：数据迁移至云开发数据库或后端 API
