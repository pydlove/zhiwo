# 博主创作平台设计文档

**日期**: 2026-04-14  
**状态**: 需求已确认，待实现规划  
**范围**: MVP（最小可行产品）

---

## 1. 项目背景

原项目为小程序形态（uni-app + 微信原生），现调整为面向文章创作者的 **Web 平台**，分为 **用户端** 和 **管理端**。平台核心逻辑是通过聚合公众号/今日头条等平台的赛道数据、头部博主及爆款文章，帮助创作者快速找到创作方向，并借助 AI（Kimi）完成大纲或全文创作。

---

## 2. 设计目标

- 让用户通过「选赛道 → 看博主 → 读爆款 → 得标题 → AI 创作」的闭环，降低选题和写作门槛。
- 让运营人员通过管理后台高效维护赛道、博主、文章等核心资产。
- 全局可配置 AI 接口（Kimi API key），创作内容支持样式模板和 Word 导出。

---

## 3. 用户端 Web

### 3.1 登录

- **账号来源**: 用户无自主注册入口，由管理员在管理后台统一开户后分配账号和初始密码
- **登录页**: 支持手机号/邮箱/用户名 + 密码登录，需输入图形验证码，防止暴力破解攻击
- **权限控制**: 未登录用户可浏览首页和赛道详情，但点击「AI 生成创作方向」或进入「创作中心」时强制跳转登录页

### 3.2 首页 · 数据看板 + 赛道入口

- **顶部导航**: Logo、首页、创作中心、我的草稿、帮助文档（未登录显示「登录」按钮；已登录显示头像/用户名下拉菜单，含个人中心、密码设置、退出登录）
- **平台切换器**: 顶部导航常驻平台切换（公众号 / 今日头条 / 百家号），默认选中公众号
- **数据看板**: 展示平台核心数据卡片
  - 今日新增爆款文章数
  - 累计赛道数
  - 累计头部博主数
  - 累计爆款文章数
- **赛道列表**: 卡片式网格布局，展示热门赛道及博主数量，点击进入赛道详情

### 3.3 赛道详情页 · 博主 + 爆款文章

- **左侧栏**: 该赛道下的头部博主列表，点击切换查看不同博主
- **右侧区**: 当前博主的热门爆款文章列表
  - 每篇文章展示：标题、阅读量/互动数据、摘要
  - 点击文章卡片可弹出全文预览（类似公众号阅读层）
  - 提供「AI 生成创作方向」按钮
- **AI 标题生成**: 调用 Kimi API，基于当前赛道 + 博主 + 爆款文章的内容特征，实时生成推荐标题
- 用户点击「AI 生成创作方向」后，进入「今日推荐创作方向」页选择标题

### 3.4 今日推荐创作方向

- 用户从赛道详情页点击「AI 生成创作方向」后进入
- 页面展示基于当前赛道/博主生成的 5 个推荐标题，每个标题带标签（推荐、高潜力、热点结合、共鸣型、实用型）
- 用户点击标题后弹出确认弹窗，确认后进入创作页
- 标题为系统每日统一生成推荐，不支持手动重新生成

### 3.5 创作页 · 双栏工作台

采用 **混合模式**（首页流程引导 + 创作页双栏效率布局）：

- **左侧参考库**（约 30-35% 宽度）
  - 显示当前选中的「推荐标题」
  - 显示来源信息：赛道 / 博主 / 参考文章
  - 显示参考文章摘要或原文片段
- **右侧编辑区**（约 65-70% 宽度）
  - **独立标题输入框**: 文章标题与正文编辑区分离，支持手动修改
  - **富文本工具栏**: 加粗、斜体、H1/H2、无序列表、有序列表、引用、链接
  - **创作模式切换**:
    - **半人工创作**: AI 根据标题生成文章大纲
    - **零人工创作**: AI 根据标题直接生成完整文章全文
  - **实时预览**: 编辑器内容与最终样式同步
  - **样式模板切换**: 工具栏显示当前模板，支持快速切换
  - **保存草稿**: 手动保存，保存后在顶部导航「我的草稿」和个人中心均可查看
  - **导出 Word**: 一键导出 `.docx`，内容按当前样式模板渲染

### 3.6 个人中心

弹框形式，标签页切换：

- **创作记录**: 历史创作记录列表（标题、创作时间、模式、操作：继续编辑 / 删除 / 导出）
- **样式模板**: 内置模板卡片选择，带实时预览。选中模板后创作和导出均按该模板渲染
- **密码修改**: 支持修改原密码、新密码、确认新密码

### 3.7 草稿查看

- 顶部导航常驻「我的草稿」入口
- 保存草稿后弹出 Toast 提示，可直接点击查看草稿
- 草稿和已完成文章统一在「创作记录」中展示

## 4. 管理端 Web

管理端延续现有桌面后台风格（Ant Design），采用左侧导航 + 右侧内容区的经典布局。

### 4.1 菜单结构

- 仪表盘
- 赛道管理
- 博主管理
- 文章管理
- 用户管理
- 系统配置

### 4.2 各模块功能

**赛道管理**
- 赛道的增删改查
- 字段：赛道名称、所属平台（公众号/今日头条/...）、简介、封面图、排序

**博主管理**
- 博主的增删改查
- 字段：博主名称、头像、所属赛道、平台主页链接、粉丝数/影响力数据
- 支持绑定多个赛道（若业务需要，MVP 先按单一赛道处理）

**文章管理**
- 爆款文章的增删改查
- 字段：文章标题、作者（博主）、所属赛道、原文链接、阅读量、摘要/正文、发布时间
- 支持按赛道、博主筛选和搜索

**用户管理**
- 查看用户列表，支持按用户名/手机号搜索
- **管理员开户**: 创建新用户账号，设置初始密码、授权到期时间
- **账号操作**: 重置密码、禁用/启用账号、修改到期时间
- 查看用户的创作历史记录
- **授权过期控制**: 到期账号无法登录，登录时系统校验有效期

**系统配置**
- Kimi API key 配置
- 平台名称、Logo 等基础配置
- 预留其他 AI 模型配置项

## 5. 数据模型（核心实体）

```
Track（赛道）
  ├── id
  ├── name
  ├── platform_type        // 公众号 / 今日头条 / 其他
  ├── description
  ├── cover_image
  ├── sort_order
  └── created_at

Blogger（博主）
  ├── id
  ├── name
  ├── avatar
  ├── track_id             // MVP 阶段先绑定单一赛道
  ├── platform_url
  ├── follower_count
  └── created_at

Article（文章）
  ├── id
  ├── title
  ├── blogger_id
  ├── track_id
  ├── source_url
  ├── read_count
  ├── summary / content
  ├── published_at
  └── created_at

User（用户）
  ├── id
  ├── username / phone / email
  ├── password_hash
  ├── created_at
  └── updated_at

CreationRecord（创作记录）
  ├── id
  ├── user_id
  ├── title                // AI 生成的创作标题
  ├── mode                 // outline / fulltext
  ├── content              // 最终创作内容（Markdown/HTML）
  ├── style_template       // JSON 样式配置快照
  ├── ref_article_id       // 参考文章 ID
  ├── ref_blogger_id
  ├── ref_track_id
  └── created_at / updated_at

StyleTemplate（样式模板）
  ├── id
  ├── user_id
  ├── name
  ├── font_family
  ├── font_size
  ├── line_height
  ├── heading_styles       // JSON
  └── created_at
```

## 6. 核心流程

### 6.1 用户创作流程

1. 用户进入首页 → 查看数据看板 → 选择赛道
2. 进入赛道详情 → 左侧选博主 → 右侧看爆款文章（可点击看原文）
3. 点击「AI 生成创作方向」→ 若未登录则跳转登录页，已登录则进入「今日推荐创作方向」页
4. 在推荐标题列表中选择标题 → 确认后进入创作页
5. 选择创作模式（半人工/零人工）→ 调用 Kimi API 生成大纲或全文
6. 用户在编辑器中修改内容（富文本工具栏 + 独立标题输入框）→ 实时预览样式效果
7. 切换样式模板（可选）
8. 导出 Word 或保存草稿（保存后可通过「我的草稿」或 Toast 快捷查看）

### 6.2 AI 调用点

| 调用点 | 输入 | 输出 | 说明 |
|--------|------|------|------|
| 生成标题 | 赛道名称 + 博主信息 + 参考文章摘要 | 3-5 个推荐标题 | 实时生成，不持久化 |
| 生成大纲 | 标题 + 赛道/博主背景 | 结构化大纲（Markdown 列表） | 半人工模式 |
| 生成全文 | 标题 + 大纲（如有）+ 赛道/博主背景 | 完整文章正文 | 零人工模式 |

## 7. 技术架构

- **后端**: Spring Boot（复用现有 `backend` 项目）
- **用户端前端**: 新建 Web 项目（Vue 3 或 React），响应式适配桌面端优先
- **管理端前端**: 复用现有管理后台或新建 Web 项目（现有为 Ant Design 风格）
- **数据库**: MySQL（复用现有 `db` 目录中的结构或重新设计）
- **AI 服务**: Moonshot（Kimi）API，通过后端代理调用，API key 由管理端配置
- **文件导出**: 后端集成 docx 生成库（如 Java 的 `docx4j` 或其他方案）

---

## 8. 关键规则与约束

- **数据录入**: MVP 阶段以手动录入为主，表结构预留扩展字段，后续可接入自动采集。
- **标题生成**: 不由管理端维护标题库，全部由 AI 根据现有数据实时生成。
- **权限**: 用户端需登录后才能进入创作页；管理端需独立管理员账号。
- **样式模板**: MVP 内置多套模板供用户选择，创作和导出均按选中模板渲染。

---

## 9. 明确排除在 MVP 外的功能

- 付费会员 / 积分体系
- 多平台一键发布（公众号/头条自动发文）
- 自动采集爬虫（仅预留扩展）
- 用户自定义模板（仅支持系统内置模板选择）
- 团队协作 / 多用户共享素材
